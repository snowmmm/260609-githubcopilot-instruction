# API 仕様

## HTTP エンドポイント

### `GET /health`

サーバーの稼働状態を確認するヘルスチェックエンドポイント。

**レスポンス**

```json
{ "status": "ok" }
```

---

## WebSocket インターフェース

### 接続先

```
ws://localhost:3001
```

クライアント接続時、サーバーは自動的に `CopilotClient` セッションを作成します。セッション作成に失敗した場合は `error` メッセージを送信して接続を閉じます。

---

### サーバー → クライアント メッセージ

| `type`      | ペイロード                                              | 説明                                         |
|-------------|--------------------------------------------------------|----------------------------------------------|
| `connected` | `{ type: "connected", sessionId: string }`             | Copilot セッション作成完了。セッション ID を通知。 |
| `delta`     | `{ type: "delta", content: string }`                   | ストリーミング中の応答チャンク。                |
| `message`   | `{ type: "message", content: string }`                 | アシスタントの完全な応答メッセージ。            |
| `idle`      | `{ type: "idle" }`                                     | セッションがアイドル状態に移行（応答完了）。    |
| `error`     | `{ type: "error", message: string }`                   | エラー発生通知。                               |

---

### クライアント → サーバー メッセージ

| `type` | ペイロード                               | 説明                        |
|--------|------------------------------------------|-----------------------------|
| `chat` | `{ type: "chat", content: string }`      | ユーザーのチャットメッセージ。 |

**例**

```json
{ "type": "chat", "content": "こんにちは！" }
```

---

## 環境変数

| 変数名  | デフォルト値   | 説明                                    |
|---------|----------------|-----------------------------------------|
| `PORT`  | `3001`         | サーバーがリッスンするポート番号。        |
| `MODEL` | `gpt-5-mini`   | Copilot セッションで使用するモデル名。    |
